## üí° IMPLEMENTATION TIPS FROM EXPERIENCE

### From 20 Years of Android Development

**Do:**
‚úÖ Read architecture first (don't jump to code)  
‚úÖ Write tests as you code (not after)  
‚úÖ Profile on real devices (not emulator)  
‚úÖ Follow clean architecture patterns  
‚úÖ Document as you implement  
‚úÖ Code review before merging  
‚úÖ Test all API levels  

**Don't:**
‚ùå Skip the architecture section  
‚ùå Implement without tests  
‚ùå Optimize before profiling  
‚ùå Skip error handling  
‚ùå Use memory leaks as features  
‚ùå Ignore performance guarantees  
‚ùå Deploy without verification  

### Performance Optimization Tips
```
1. Cache aggressively (500ms TTL is proven)
2. Use Flow, not callbacks (backpressure!)
3. Profile early and often
4. Test on multiple device generations
5. Monitor battery impact closely
6. Keep history size bounded (300 items)
```

### Testing Tips
```
1. Write unit tests for business logic
2. Mock external dependencies
3. Test edge cases explicitly
4. Integration test the complete flow
5. Performance test on low-end device
6. Check for memory leaks (LeakCanary)
```
